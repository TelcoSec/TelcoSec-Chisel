TelcoSec-Chisel

Telecom Security Auditing & Baseband Analysis Platform

TelcoSec-Chisel is a bootable Live Linux distribution purpose-built for cellular security researchers, Software Defined Radio (SDR) engineers, and hardware pentesters. Based on Ubuntu 24.04 LTS (Noble Numbat) with a customized, low-overhead XFCE desktop environment, it packs pre-configured drivers, simulation frameworks, protocol dissectors, and smartcard utilities.

OS Customizations & Tuning

Lowering latency and unlocking kernel interfaces for telecom auditing

Telecom software suites (like srsRAN, OAI, or SigPloit) place extreme demands on kernel timers, socket memory buffers, and transceiver streaming rates. TelcoSec-Chisel applies specific kernel and PAM optimizations by default.

1. Real-time Scheduling (PAM & Groups)

Software Defined Radios require consistent scheduling intervals to avoid sample drops (under/overruns). Under standard Linux settings, non-root users cannot request real-time execution priority.

/etc/security/limits.d/99-realtime.conf
# Enable real-time scheduling priority up to 99 and unlimited locked memory
@realtime       -       rtprio          99
@realtime       -       memlock         unlimited

The default user telcosec belongs to the system realtime group, enabling GNU Radio and srsRAN threads to lock samples into physical RAM and run at scheduling priority 99.

2. Kernel SCTP Stack Optimizations

SCTP (Stream Control Transmission Protocol) is the transport layer backbone of telecom networks (SS7/M3UA, Diameter over SCTP, S1AP, and NGAP). Standard OS kernels are optimized for TCP/UDP. TelcoSec-Chisel preloads the kernel SCTP module and tunes socket limits in /etc/sysctl.d/99-sctp-tuning.conf:

SCTP sysctl parameters
# Increase buffer limits for telecom signaling
net.sctp.sctp_mem = 94500000 915000000 927000000
net.sctp.sctp_rmem = 4096 87380 8388608
net.sctp.sctp_wmem = 4096 16384 8388608

# Tuning retransmission timeouts (prevent scanner hangs)
net.sctp.rto_min = 200
net.sctp.rto_max = 800
net.sctp.association_max_retrans = 4
net.sctp.path_max_retrans = 2

3. Low-Latency USB & GRUB Configurations

To run high-bandwidth 5G NR Rogue Base Stations, USB polling jitter must be eliminated. The system deploys udev rules to turn off autosuspend delay on USRP B210 and HackRF hardware:

/etc/udev/rules.d/51-usb-latency.rules
ACTION=="add", SUBSYSTEM=="usb", ATTR{idVendor}=="2514", ATTR{power/autosuspend_delay_ms}="0"
ACTION=="add", SUBSYSTEM=="usb", ATTR{idVendor}=="2514", ATTR{power/control}="on"
ACTION=="add", SUBSYSTEM=="usb", ATTR{idVendor}=="1d50", ATTR{power/control}="on"

Additionally, GRUB configs disable CPU mitigations (mitigations=off) and set the clocksource to TSC for stable timing on bare metal installations.

Drivers & Hardware Access

Plugging in transceivers, readers, and sniffers without root

Traditional security environments force you to run RF interfaces as root to access raw USB descriptors. TelcoSec-Chisel utilizes custom udev permissions, allowing members of the standard `plugdev` group to read and write directly to transceivers.

1. Non-Root Hardware Access Rules

The file /etc/udev/rules.d/50-telcosec-hw.rules maps USB product and vendor codes, assigning ownership group to `plugdev` and opening permissions:

udev Vendor / Product Maps
# RTL-SDR Dongles
SUBSYSTEMS=="usb", ATTRS{idVendor}=="0bda", ATTRS{idProduct}=="2832", MODE:="0666", GROUP:="plugdev"
# HackRF One
ATTR{idVendor}=="1d50", ATTR{idProduct}=="6089", SYMLINK+="hackrf-%k", MODE:="660", GROUP:="plugdev"
# Ettus USRP B-Series
ATTRS{idVendor}=="2514", ATTRS{idProduct}=="0002", MODE:="0660", GROUP:="plugdev"
# Osmocom SIMtrace 2 sniffer
ATTRS{idVendor}=="1d50", ATTRS{idProduct}=="60e3", MODE:="0660", GROUP:="plugdev"

2. Software Defined Radio (SDR) Drivers Sandbox

GNU Radio and hardware transceiver drivers are isolated in a custom Conda environment named telcosec-sdr. This prevents host apt-package scripts or pip executions from breaking driver configurations.

To execute commands that require UHD, HackRF, LimeSDR, or SoapySDR bindings, activate the virtual environment:

Activate Conda Environment
# Activate sandbox
conda activate telcosec-sdr

# Verify SoapySDR driver bindings are visible
SoapySDRUtil --find

3. Smartcard PC/SC Readers (SIM Auditing)

Auditing physical smartcards requires communicating with USB smartcard adapters. TelcoSec-Chisel preconfigures the standard PC/SC daemon (pcscd). Insert your card reader (e.g. Omnikey, ACS) and run:

Query Smartcard Readers
# Check daemon state
sudo systemctl status pcscd

# List connected card readers and cards
pcsc_scan

5Ghoul Fuzzer Guide

Fuzzing 5G NR mobile modems on TelcoSec-Chisel

5Ghoul is a 5G NR baseband fuzzer. It uses an OpenAirInterface (OAI) rogue base station implementation to transmit malformed RRC, NAS, and MAC messages over the air, uncovering vulnerabilities in smartphones, baseband modems, and IoT modules.

Compile Deferred to First Run
To avoid inflating the ISO size and build times (compiling OAI libraries takes over 30 minutes), the framework dependencies are pre-installed in the OS, but the cloning and compiling steps are deferred to first execution. **Do not run 5Ghoul inside a Virtual Machine; it requires native USB 3.0 throughput to stream RF.**

Deploying 5Ghoul on your Live System

1
Execute the Installer
Run the installer helper. Provide the radio backend parameter to target either your USRP B210 (default) or a BladeRF micro A4.
Compile for Hardware
# Option A: Compile for USRP B210
sudo 5ghoul-install

# Option B: Compile for BladeRF A4 (performs patch translations)
sudo 5ghoul-install --radio BLADERF
2
Start the Core Network
5Ghoul simulates the radio interfaces, but requires an active mobile core database to process cellular attachment. Launch Open5GS services and add the default test credentials to the network subscriber database:
Setup 5G Core Subscriber
# Start local 5G SA core functions
sudo open5gs-start

# Register test subscriber profile
sudo 5ghoul-add-subscriber
3
Run the Fuzzer
Connect your SDR to a USB 3.0 port on the host, place the test smartphone inside an RF shield box connected via coaxial cable, and boot the fuzzer.
Launch Fuzzer Engine
# Run NAS fuzzer suite
sudo 5ghoul-run --Attack.Name=NAS_5GS_Fuzz --UE.IMSI=001011234567890

Developer Guide

Building the Live ISO from scratch

TelcoSec-Chisel uses an automated bash orchestration build chain to bootstrap, configure, and output bootable XFCE live desktop images.

1. Setup Compilation Host

The build environment must be run on a native Ubuntu or Debian host machine (or inside a WSL2 container with systemd enabled). You need approximately 20 GB of free storage.

Install Host Dependencies
# Update and install build packages
sudo apt-get update
sudo apt-get install -y debootstrap squashfs-tools grub-pc-bin grub-efi-amd64-bin xorriso mtools zstd

2. Run the Build Pipeline

Clone this repository, review configurations inside builder/, and execute the master build script.

Compile Live ISO
# Execute compilation
sudo ./build-iso.sh

3. Pipeline Phase Execution Order

The orchestrator bootstraps a minimal chroot container and runs provisioning scripts sequentially. Do not alter their index prefixes, as each step relies on outputs of the previous scripts:

Script File Provisioning Operation
00-install-all-packages.sh Pre-downloads all apt packages to speed up provisioning.
01-install-base.sh Installs XFCE, LightDM, Firefox, base compilers, and sets live boot user.
02-install-sdr.sh Creates Conda environment; compiles UHD, HackRF, and BladeRF SDR drivers.
03-install-core-network.sh Compiles and sets up the srsRAN simulation suite.
04-install-tools.sh Deploys Wireshark, SigPloit, Diafuzzer, SIPVicious, Scapy, softphones, and dictionaries.
05-desktop-customization.sh Sets desktop custom wallpapers, icons, Firefox bookmarks toolbar, and start documentation page.
06-install-ue-analysis.sh Deploys FirmWire baseband QEMU environment, MobileInsight, QCSuper, and card programming scripts.
07-install-installer.sh Integrates the Calamares installer engine with customized TelcoSec branding graphics.
08-system-optimization.sh Injects the low-latency udev configs, SCTP limits, CPU isolation limits, and CA certificates.
09-install-5ghoul.sh Deploys the 5Ghoul ogstun virtual network adapters and compilation frameworks.
